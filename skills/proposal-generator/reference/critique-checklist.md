# Critique checklist — roasting a client's existing quotes/proposals

Use before building, when the client shares their current proposals. The goal: diagnose
why the current document competes on price, and what real substance to carry into the new one.

## The core reframe: quote vs proposal
Most firms send a **QUOTE** (answers "what + how much") and call it a proposal. A real
**PROPOSAL** answers "**why you, why us, why now, what's the risk** — *then* what + how much".
If the document leads with scope + exclusions + a total, it's a quote → the buyer compares on
price → commoditised. Name this first.

## Production / credibility red flags
- [ ] **Exported from a spreadsheet?** Excel→PDF leaks cell order (totals float, labels detach
      from values) and often shows the `.xlsx` filename / "page x of y" in the footer. Looks
      internal, not client-facing. (For a firm whose value is "clean & precise", a messy quote
      is a credibility own-goal.)
- [ ] **Zero brand** — no logo styling, colour, typography, hierarchy.
- [ ] **Copy-paste errors** — wrong project name in the title, repeated/garbled blocks, typos.
- [ ] Inconsistent header/descriptor across documents.

## Sales / content red flags
- [ ] **All scope + exclusions, no value** — no who-we-are framed for the buyer, no proof, no
      "why us", no method.
- [ ] **Exclusions outnumber inclusions** — the dominant message becomes "what we WON'T do".
      Necessary legally, but must be subordinated (move to an FAQ/"scope boundaries" framing).
- [ ] **No proof / risk-reversal** — track record, QA, insurance, standards absent (usually the
      firm's STRONGEST assets, missing from the actual sales doc).
- [ ] **Weak/absent CTA + no validity date** — ends on "Total + Raised by [Director]".
- [ ] **Inside-out** — about scope, not the buyer's problem/context.
- [ ] **One price, no options** — nothing to anchor; easy to haggle. (Fix: 2-3 tiers.)
- [ ] **Inconsistent pricing presentation / VAT** across documents.

## What's usually worth KEEPING
- The engineering/technical substance (precise scope, sensible exclusions) — plugs straight
  into the new Scope + Pricing sections.
- Any per-line/per-level fee breakdown — that's the model to standardise (it anchors value).

## Output of the critique
1. The quote-vs-proposal verdict. 2. Prioritised gap list. 3. What to keep. 4. The ICP /
work-type axis revealed by the examples (e.g. for a BIM/MEP contractor: detailed-design
modelling / point-cloud existing-conditions / coordination — the 3-block swap dimension). Then build.
