# Send-proposal structure (variant B) — the self-contained document

The send proposal travels alone (email / follow-up / forwarded internally). It must sell
WITHOUT a human in the room — so it carries the "why" in writing that the call-deck would
carry by voice. Render as a scrolling single-page document on the client's brand, with a
sticky top bar (wordmark + CTA) and a scroll progress bar. ~A4-width column (~820px).

## Anatomy
1. **Cover** — dark brand panel: "Proposal · prepared for [Contact], [Company]", a headline
   value prop, sub-line, meta (prepared-for / project / date · "valid 30 days").
2. **Summary** — the short version: who the client is, what this proposal covers, the outcome.
   End with a compact trust strip (the proof numbers).
3. **The situation** (problem) — 3 named pains with their on-site / business consequence.
4. **Our approach** — the client's signature method explained (the framework), so "no
   surprises" reads as structural, not a promise.
5. **Scope — what you get** — clean checklist of deliverables (this is where the real scope
   from their existing quotes plugs in).
6. **Proof** — 2 case cards (matched sector) with exact result numbers + one positioning quote.
7. **Why [client]** — the differentiators (dedicated team / white-label / insurance / standards).
8. **Investment** — a pricing **table** (Package / Scope / Turnaround / Investment) with 2-3
   packages + the **PDF cost-breakdown button** + an "all packages include…" reassurance line.
9. **FAQ / objections** — pre-handle the 4-6 real objections (why-not-in-house, is-white-label-
   invisible, quality/liability, speed, scope-change). This is the send-doc's version of the
   call-deck's "Alternatives" + live Q&A.
10. **How we start** — 3 numbered steps + primary CTA + a signature block (name, title, contact).
Footer: "valid 30 days from [date]". Add `@media print` so it exports cleanly to PDF.

## Differences vs the call-deck
- Pricing is a **table** (reads alone), not visual tier-cards.
- Has a **FAQ/objections** section + a fuller **Scope** list (the rep isn't there to answer).
- Copy is heavier and complete; the deck's copy is deliberately sparse.
- Same brand, same proof, same placeholders, same PDF cost-breakdown button.
